📋 Phase 1: Project Foundation

🎯 Objective

Establish the foundational requirements and standards for OCED-Sprinklr-Viber integration.

📝 Key Activities

1
Define Current Focus
Set up integration specifically for Viber profiles within Sprinklr ecosystem
2
Identify Data Source
Confirm OCED team as primary data provider and establish governance protocols
3
Specify Data Format
CSV with ~20 headers, semicolon delimiter, UTF-8 encoding
4
File Naming Convention
Standard Format: OCED BYOC_Sprinklr Viber template
Required Headers Include: MDM ID Key, Topic Name, Segment ID, Segment Name, Viber Consent Setting Code, Channel, and 14+ additional headers as per OCED specification.
⚠️ Important Notes: Ensure data privacy compliance throughout the process. Maintain audit trails for all data processing activities. Coordinate with legal/compliance teams for approval.
⚙️ Phase 2: Technical Configuration

🎯 Objective

Configure Sprinklr data connectors and custom fields to enable automated OCED data processing.

🔧 Prerequisites

  • Sprinklr administrator access
  • OCED SFTP server credentials
  • Technical team availability

Configuration Steps

1
Access Sprinklr Configuration Area
Navigate to Settings → Data Connectors with administrator privileges
2
Create Data Connectors
Set up Weekly Connector (primary) and Ad-hoc Connector (secondary)
3
Configure SFTP Connection
Set server details, authentication, directory paths, and security settings
4
Configure Field Mapping
Map OCED headers to Sprinklr custom fields using naming convention
5
Configure Scheduling
Set continuous monitoring (24/7) until OCED provides specific timing

🎯 Phase 7: Audience Targeting Method 1 - Filter-Based

📊 Filter-Based Targeting (Dynamic Approach)

Enable real-time, flexible audience targeting using dynamic filters without development dependencies.

👥 Target Users

Marketing Managers, Campaign Specialists, Content Strategists

Process Overview

1
Identify Base Audience
All Viber profiles under "Audience Profiles" in "OCED all profile" list
2
Clone Journey Template
Select and clone the filter-based Journey template
3
Configure Decision Node
Edit Decision Box with Topic Name, Segment ID/Name, and Consent filters
4
Save Configuration
Marketing Independence: Direct filtering without technical setup
Marketing Independence: Marketers can directly filter on new topic/segment values as they appear in the data, without requiring prior setup by the Sprinklr technical team.

💬 Phase 9: Message Content Configuration

Message Content Setup

Configure message content using direct nodes or DAM templates (applicable to both targeting methods).

Option A: Message Node
  • Select/Add "Send Message" node
  • Configure Channel: Select Viber
  • Configure Account: Select Infobip account
  • Configure Content: Direct text input with personalization
Option B: DAM Template Node
  • Create DAM Asset in Digital Asset Manager
  • Input message content into DAM template
  • Submit for review/approval
  • Select DAM template in journey node

🔄Decision Framework

Choose Method 1 (Filter-Based) When:
  • Urgent campaign requirements
  • New segment targeting
  • Experimental campaigns
  • High flexibility needed
Choose Method 2 (Profile List-Based) When:
  • Established campaign patterns
  • Quality assurance priority
  • Organizational documentation needs
  • Recurring campaign series
